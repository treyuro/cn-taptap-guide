# cn-taptap-guide

## 项目简介

**cn-taptap-guide** 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，而是作为一个通用的页面集合，存放与特定主题或功能相关的静态 HTML 文件。

这些页面可能包含信息汇总、工具界面、文档导航或其他形式的静态内容，旨在以简洁的 HTML 形式提供可访问的参考材料。

## 目录说明

```
cn-taptap-guide/
├── pages/           # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/          # 页面引用的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── index.html       # 仓库首页或导航页（如有）
└── README.md        # 本文件
```

- `pages/`：所有独立 HTML 页面均放置在此目录下，每个页面为一个完整的、可独立浏览的文档。
- `assets/`：存放页面所需的公共资源文件，按类型组织。
- `index.html`：可选的入口页面，用于快速导航或展示概览。

## 页面归档说明

本仓库内的 HTML 页面均为静态文件，不依赖后端服务或数据库。每个页面在归档时已确保内容完整，可离线或在线直接打开浏览。

归档原则：
- 页面内容保持原样，不进行额外修改或优化。
- 不包含任何动态追踪、广告或外部请求（除非页面本身需要）。
- 文件名和目录结构尽量保持清晰，便于检索。

## 维护说明

本仓库由社区维护，欢迎提交 Issue 或 Pull Request 以补充、修正或更新页面内容。

维护注意事项：
- 新增页面请放入 `pages/` 目录，并确保路径引用正确。
- 若页面使用外部资源，请尽量将资源文件放入 `assets/` 目录并引用本地路径。
- 请勿添加任何违反平台政策或法律法规的内容。
- 提交前建议预览页面，确保功能正常。

## 使用方式

您可以直接通过 GitHub Pages 或其他静态托管服务部署本仓库，即可在线访问所有归档页面。也可以将仓库克隆到本地，在浏览器中打开 `pages/` 下的任意 HTML 文件。
